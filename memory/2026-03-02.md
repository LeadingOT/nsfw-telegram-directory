## [P0] Astro 5 Content Collections Issue发现

**问题:**
- Astro 5.17.3的content collections无法识别glossary collection
- 即使使用glob loader也无法加载JSON files
- listings collection工作正常，但glossary不工作

**调试过程:**
- 尝试1: 使用`type: 'data'` - 失败
- 尝试2: 使用glob loader with `*.json` pattern - 失败
- 尝试3: 使用glob loader with `**/*.json` pattern - 失败
- 尝试4: 重命名collection from 'glossary' to 'terms' - 失败
- 尝试5: 删除config.ts (让Astro auto-detect) - glossary仍失败，但listings工作
- 尝试6: 创建小test文件 - 失败
- 尝试7: 把glossary文件复制到listings directory - 成功（说明不是文件问题）

**解决方案:**
- 放弃content collections for glossary
- 使用直接JSON import approach
- 在pages里hardcode所有10个import statements
- 手动构建static paths

**Trade-offs:**
- ✅ Pro: 立即工作，no更多debugging
- ✅ Pro: Build速度快 (1.79s)
- ✅ Pro: 完全控制data flow
- ❌ Con: Adding new terms需要手动更新import list
- ❌ Con: 不是scalable solution (如果100+术语)

**For future:**
- 如果glossary扩展到20+术语，考虑:
  - 升级Astro到更新版本
  - 使用glob() in getStaticPaths instead of imports
  - 或者用Node.js fs.readdir动态读取JSON files
