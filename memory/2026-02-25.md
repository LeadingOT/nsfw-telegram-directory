## 🚀 RoleOffer.com 项目启动（重大决策）

### 决策背景
- Directory Factory已稳定运行（12站，1,376页，自动化流程完善）
- 需要多元化收入来源，探索SaaS模式
- 发现市场机会：Pave/Carta收费$10-25K/年，可做$49-99的轻量级替代品

### 产品定义
**RoleOffer.com** - Startup Compensation Tool

**核心价值：**
- 轻量级，30秒生成完整offer package
- 按offer收费$49-99（vs Pave $10-25K/年）
- 不需要用户share数据（vs Pave/Carta需连接HRIS）
- 目标用户：Seed/A/B轮startup founder，5-200人，没有HR team

**输入：**
- Role (e.g., Software Engineer, Product Manager)
- Level (IC1-IC6/M1-M3)
- Location (San Francisco, New York, etc.)
- Company Stage (Seed/A/B/C)
- Industry (optional)

**输出（4个核心功能）：**
1. **Comp Benchmark Report** - Base salary P25/P50/P75, Equity range, Total comp
2. **Offer Letter PDF** - 可下载的professional offer letter（6个预设模板）
3. **Equity Value Projection** - 1yr/2yr/4yr价值，3个scenario (conservative/base/optimistic)
4. **Negotiation Talk Points** - AI生成的founder cheat sheet

### Tech Stack
- **Frontend:** Next.js 14 + Tailwind + shadcn/ui
- **Backend:** Supabase (database + auth + storage)
- **AI:** GPT-4o-mini (offer letter生成 + talk points)
- **PDF:** React-PDF or Puppeteer
- **Payment:** Airwallex (不是Stripe)
- **Deployment:** Vercel

### 域名
- **roleoffer.com** (Bill已注册，2026-02-25)
- **为什么选这个名字：** pSEO完美匹配
  - "role"是20,000个pSEO页面的核心维度

  - 比OfferDraft更SEO友好（role直接在URL里）

### pSEO策略（关键获客渠道）
**规模：** 20,000+页面

**URL结构：**
```
/compensation/{role}-{level}-{stage}-{location}
例如：/compensation/senior-software-engineer-series-a-san-francisco
```

**页面矩阵：**
- 50 roles × 5 levels × 4 stages × 20 cities = 20,000组合

**每页内容（自动生成）：**
- H1: "{Role} Compensation at {Stage} Startups in {Location} (2026)"
- Salary range chart (P25/P50/P75)
- Equity range + vesting info
- Total comp估算
- vs FAANG对比
- CTA: "Generate a complete offer package → $49"
- FAQ section (schema markup for rich snippets)

**Phase 1实施：**
- 先生成top 5,000页（高流量role × location组合）
- 使用Next.js ISR (Incremental Static Regeneration)
- 每周扩展500-1,000新页面

### 数据层（全免费数据源）
1. **H1B Salary Data** (h1bdata.info) - Base salary by role/company/location
2. **Salary Transparency Job Postings** - Scrape Indeed/LinkedIn (CA/CO/NY/WA必须公开salary)
3. **levels.fyi** - Tech company公开comp数据
4. **BLS API** - Bureau of Labor Statistics公开API
5. **Equity Benchmarks** - SaaStr/Kruze/Carta公开报告

**数据处理：**
- 统一role taxonomy (不同title mapping到标准level)
- 多源加权平均 → P25/P50/P75 percentile bands
- 存储：role × level × location × stage
- 每周自动更新（cron job scrape + ETL）

**Supabase Schema设计：**
```sql
-- roles: id, title, category, level
-- locations: id, city, state, country
-- stages: id, name (Seed/A/B/C)
-- comp_data: role_id, level, location_id, stage_id, p25, p50, p75, equity_pct, updated_at
```
